## 🏗️ 硬件配置

### 主控芯片
- **型号**: STM32F407ZGT6
- **封装**: LQFP144
- **内核**: Cortex-M4 (带FPU)
- **时钟**: 168MHz (HSE=25MHz, PLL配置)
- **内存**: 
  - RAM: 192KB (0x20000000-0x2001FFFF)
  - Flash: 1MB (0x8000000-0x80FFFFF)

### 外围设备配置
1. **振荡器**: HSE外部晶体 (PH0/PH1)
2. **调试接口**: SWD (PA13/PA14)
3. **UART接口**:
   - USART1: PA9(TX), PA10(RX)
   - USART2: PA2(TX), PA3(RX)
   - USART3: PB10(TX), PB11(RX)
4. **GPIO输出**:
   - LED0: PF9
   - LED1: PF10

## 🛠️ 软件架构

### 操作系统
- **RTOS**: FreeRTOS (CMSIS-OS V2 API)
- **任务优先级**: NVIC优先级组4 (4位抢占, 0位子优先级)

### 主要任务设计
```c
// 三任务协作模型
1. UART任务 (uart_task.c)
   - 职责: 监听USART3接收上位机指令
   - 输入: ASCII字符 '3'~'9' (出货指令)
   - 输出: 写入共享内存指令区

2. Motor任务 (motor_task.c)  
   - 职责: 轮询检查出货指令，控制电机
   - 输入: 读取共享内存指令区
   - 输出: 通过USART2发送电机控制协议

3. Status任务 (未看到具体文件，但通信协议中包含)
   - 职责: 上报状态给上位机
   - 输出: 通过USART3发送状态报告
```

### 共享内存模型
```c
typedef struct {
```

## 🔧 电机驱动板协议

### 帧格式
```
[2B帧头] [1B地址] [1B版本] [2B帧ID] [1B CMD] [1B LEN] [nB DATA] [1B DC]
  0x57     0x58     0x66     递增ID   指令码   数据长度   数据     校验
   'W'      'X'      'f'     
```

### 指令集
| 指令码 | 功能 | 说明 |
|--------|------|------|
| 0x00 | CMD_CONNECT | 握手/连接测试 |
| 0x01 | CMD_POLL_BOARD | 查询驱动板整体状态 |
| 0x02 | CMD_POLL_MOTOR | 查询电机转动状态 |
| 0x03 | CMD_DISPENSE | 出货指令 |
| 0x04 | CMD_GET_DROP_SENSOR | 获取掉货检测信号 |
| 0x05 | CMD_GET_TEMP | 获取温度 |

### 电机状态定义
```c
typedef enum {
    MOTOR_STOP        = 0,   // 电机停止
    MOTOR_RUNNING     = 2,   // 电机运行中
    MOTOR_RUNNING_END = 10,  // 电机运行结束（正常完成）
} MotorState;

typedef enum {
    ERR_NO_ERR        = 0,   // 无错误
    ERR_INVALID_SLOT  = 1,   // 无效槽位
    ERR_MOTOR_BLOCK   = 3,   // 电机阻塞
    ERR_MOTOR_SHORT   = 5,   // 电机短路
    ERR_MOTOR_TIMEOUT = 6,   // 电机超时
    ERR_NO_DROP       = 7,   // 无掉货检测
} MotorErrCode;
```

## 🔄 上位机通信协议

### 指令集合
- `'3'~'9'`: 出货货道3-9
- `'1'` 或 `'a'`: LED1 ON
- `'2'`: LED1 OFF

### 响应格式
- **出货中**: `"[Motor] Aisle X: Dispensing...\r\n"`
- **完成**: `"REPORT: AISLE_X_OK\r\n"`
- **卡货**: `"REPORT: AISLE_X_ERROR\r\n"`

## 🔄 工作流程

### 正常出货流程
```
1. 用户在小程序完成兑换码验证
2. 上位机(Orange Pi)收到兑换验证通过信号
3. 上位机通过USART3发送出货指令 '3'-'9'
4. STM32 UART任务接收指令并写入共享内存
5. Motor任务读取共享内存，解析出货指令
6. STM32通过USART2发送出货命令给电机驱动板
7. 电机驱动板控制对应货道电机运行
8. 驱动板通过传感器检测掉货信号
9. STM32轮询检查出货状态
10. 出货完成后通过USART3上报结果
```

### 状态上报机制
- **实时状态轮询**: Motor任务定时查询电机状态
- **事件触发上报**: 出货开始/完成/出错时立即上报
- **故障检测**: 电机阻塞、短路、超时等异常状态
