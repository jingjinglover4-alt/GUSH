# STM32派样机下位机项目快速摘要

## 📌 核心识别

这个项目是**HI拿智能派样机系统的硬件控制核心**，位于整个系统的最后执行层：

```
小程序(用户) → Flask服务(服务器) → Orange Pi(上位机) → STM32(下位机) → 电机驱动板(执行)
```

## 🧩 项目定位

| 角色 | 设备 | 位置 | 本项目？ |
|------|------|------|----------|
| 用户界面 | 微信小程序 | 用户手机 | ❌ |
| 后台服务 | Flask应用 | 云服务器 | ❌ |
| 上位机 | Orange Pi | 机器现场 | ❌ |
| **下位机** | **STM32F407** | **机器内部** | **✅** |

## 💡 一句话概括

**这是一个用STM32芯片控制电机执行出货操作的嵌入式系统**，它接收来自Orange Pi的串口指令，然后通过特殊协议控制电机驱动板完成派样。

## 🔌 关键能力

1. **多货道管理**: 支持最多50个货道独立控制
2. **实时监控**: 监控每个电机运行状态(运行中/停止/完成/故障)
3. **故障检测**: 电机阻塞、短路、超时、掉货失败等异常检测
4. **异步通信**: FreeRTOS任务分离接收、执行、上报三个流程

## 📊 技术规格

**芯片**: STM32F407ZGT6 (Cortex-M4, 168MHz, 1MB Flash, 192KB RAM)  
**RTOS**: FreeRTOS (CMSIS-OS V2 API)  
**串口**: USART2(给驱动板), USART3(给上位机)  
**协议**: 自定义二进制帧(WX帧头)  
**开发环境**: Keil MDK (ARMCC V5.06)  

## 🎯 关键文件

```
App/
├── shared_mem.c/h    ← 任务间数据交换中心
├── uart_task.c/h     ← 接收上位机指令
├── motor_task.c/h    ← 控制电机执行
└── vending_motor.c/h ← 电机驱动板协议
```

## 🔄 工作流程

```
1. 上位机发送 '5' 到USART3 (出货5号货道)
2. uart_task解析指令，写入shared_mem.target_aisle=5
3. motor_task轮询发现出货指令，通过USART2发送出货命令
4. 驱动板控制5号货道电机转动
5. motor_task轮询电机状态，直到完成
6. 通过uart_task向USART3上报完成状态
```

## ⚠️ 特别注意事项

1. **协议兼容性**: 与电机驱动板的协议是自定义的，修改需双方同步
2. **货道编号**: 支持1-50，但实际硬件可能只有部分
3. **波特率**: USART3固定115200用于与上位机通信
4. **响应格式**: 出货完成需返回"REPORT: AISLE_X_OK"，上位机依赖此格式
