# HI拿派样机 - Orange Pi部署手册

## 硬件信息
- 主控：Orange Pi 3B V2.1（Linux）
- 下位机：STM32F407
- 连接：USB转TTL串口（115200 8N1）
- 货道：60个（A0~F9）
- 网络：WiFi（后期添加4G）

---

## 第一步：Orange Pi系统准备

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装Python和pip
sudo apt install python3 python3-pip -y

# 安装串口工具（调试用）
sudo apt install minicom -y
```

---

## 第二步：检查串口设备

插上USB转TTL后：

```bash
# 查看串口设备
ls /dev/ttyUSB*
# 应该显示 /dev/ttyUSB0

# 如果没有，检查驱动
lsusb
dmesg | tail -20

# 给当前用户串口权限（可选，root不需要）
sudo usermod -aG dialout $USER
```

用minicom测试串口通信（可选）：
```bash
sudo minicom -D /dev/ttyUSB0 -b 115200
# Ctrl+A X 退出
```

---

## 第三步：部署程序

```bash
# 创建目录
sudo mkdir -p /opt/hinana-vending
cd /opt/hinana-vending

# 复制程序文件（从你的开发电脑传过来）
# 方法1：用scp
# scp serial_controller.py inventory_manager.py local_api.py requirements.txt root@orangepi-ip:/opt/hinana-vending/

# 方法2：直接在Orange Pi上用vim/nano新建
```

```bash
# 安装Python依赖
cd /opt/hinana-vending
pip3 install -r requirements.txt
```

---

## 第四步：修改配置

编辑 `local_api.py` 顶部配置，或通过环境变量设置：

```bash
# 修改设备ID（每台机器不一样）
export DEVICE_IMEI="MACHINE-001"   # 改成你的设备编号

# 串口路径（一般不需要改）
export SERIAL_PORT="/dev/ttyUSB0"

# API端口
export API_PORT="8080"

# 安全token（改成随机字符串）
export API_TOKEN="your-secret-token-here"
```

---

## 第五步：手动测试

```bash
cd /opt/hinana-vending

# 先测试串口通信
python3 serial_controller.py

# 再启动完整服务
python3 local_api.py
```

新开一个终端，测试API：
```bash
# 健康检查
curl http://localhost:8080/health

# 查询库存
curl -H "X-Device-Token: your-secret-token-here" http://localhost:8080/inventory

# 测试出货 A0
curl -X POST \
  -H "X-Device-Token: your-secret-token-here" \
  -H "Content-Type: application/json" \
  -d '{"channel": "A0"}' \
  http://localhost:8080/dispense
```

---

## 第六步：设置开机自启

```bash
# 复制服务文件
sudo cp hinana-vending.service /etc/systemd/system/

# 修改服务文件中的环境变量（重要！）
sudo nano /etc/systemd/system/hinana-vending.service
# 修改 DEVICE_IMEI 和 API_TOKEN

# 启用并启动服务
sudo systemctl daemon-reload
sudo systemctl enable hinana-vending
sudo systemctl start hinana-vending

# 查看状态
sudo systemctl status hinana-vending

# 查看日志
tail -f /opt/hinana-vending/vending.log
```

---

## 第七步：云端配置

登录管理后台 http://150.158.20.232/admin/

1. 进入「机器管理」
2. 添加新机器，填写设备编号（与 DEVICE_IMEI 一致）
3. 在「货道管理」中配置60个货道（A0~F9）

## 常用运维命令

```bash
# 重启服务
sudo systemctl restart hinana-vending

# 停止服务
sudo systemctl stop hinana-vending

# 查看实时日志
journalctl -u hinana-vending -f

# 查看程序日志
tail -f /opt/hinana-vending/vending.log

# 查看串口设备
ls /dev/ttyUSB*

# 手动触发库存上报
curl -X POST -H "X-Device-Token: your-token" http://localhost:8080/report

# 全部货道补货
curl -X POST \
  -H "X-Device-Token: your-token" \
  -H "Content-Type: application/json" \
  -d '{"qty": 5}' \
  http://localhost:8080/inventory/replenish
```

```bash
ls /dev/ttyUSB*  # 会多出几个设备

# 安装拨号工具
sudo apt install usb-modeswitch ppp -y

# 配置网络（根据4G模块品牌和运营商配置）
# 配置完成后，程序不需要改动，自动走4G上网
```

## 文件清单

| 文件 | 说明 |
|------|------|
| `serial_controller.py` | 串口通信模块（与STM32通信） |
| `inventory_manager.py` | 库存管理模块（本地计数+云端上报） |
| `local_api.py` | HTTP API服务（主程序） |
| `requirements.txt` | Python依赖列表 |
| `hinana-vending.service` | systemd服务配置 |
| `STM32_protocol.md` | STM32固件协议说明 |
| `inventory.json` | 本地库存数据（自动生成） |
| `vending.log` | 运行日志（自动生成） |
